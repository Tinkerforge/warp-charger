# Ladezeit- und -energielimits
Mit dem `charge_limits`-Modul können Energie- und Zeitlimits definiert werden, nach denen ein Ladevorgang beendet wird. Es können sowohl Standardlimits gesetzt werden ([`charge_limits/default_limits`](/interfaces/mqtt_http/api_reference/charge_limits.mdx#charge_limits_default_limits), als auch Limits für den laufenden oder nächsten Ladevorgang überschrieben werden ([`charge_limits/override_duration`](/interfaces/mqtt_http/api_reference/charge_limits.mdx#charge_limits_override_duration) und [`charge_limits/override_energy`](/interfaces/mqtt_http/api_reference/charge_limits.mdx#charge_limits_override_energy))

## `charge_limits/restart` {#charge_limits_restart_warp1}


Setzt die Ladelimits zurück, so als ob ein neuer Ladevorgang begonnen hätte, behält aber die überschriebenen Ladelimits bei. Mit dieser API und [`charge_limits/override_duration`](/interfaces/mqtt_http/api_reference/charge_limits.mdx#charge_limits_override_duration) und [`charge_limits/override_energy`](/interfaces/mqtt_http/api_reference/charge_limits.mdx#charge_limits_override_energy) kann zu einem beliebigen Zeitpunkt ein neues "absolutes" Ladelimit gesetzt werden.
**Leerer Payload. Es muss einer der folgenden Werte übergeben werden: `null`, `""`, `false`, `0`, `[]` oder `{}`**

    <Tabs groupId="apiType" queryString className="hidden-tabs">
        <TabItem value="http" label="HTTP (curl)">
```bash
# $HOST z.B. warp-AbCd
```
#### Schreiben
```bash
curl http://$HOST/charge_limits/override_energy -d '{ "energy_wh": 25000 }'
```

oder abgekürzt:

```bash
curl http://$HOST/charge_limits/override_energy -d '25000'
```




        </TabItem>
        <TabItem value="mqtt" label="MQTT (mosquitto)">
```bash
# $BROKER z.B. my_mosquitto.localdomain
# $PREFIX z.B. warp/AbCd
```
#### Schreiben
```bash
mosquitto_pub -h $BROKER -t $PREFIX/charge_limits/override_energy -m '{ "energy_wh": 25000 }'
```

oder abgekürzt:

```bash
mosquitto_pub -h $BROKER -t $PREFIX/charge_limits/override_energy -m '25000'
```




        </TabItem>
    </Tabs>

## `charge_limits/restart` {#charge_limits_restart_any}


Setzt die Ladelimits zurück, so als ob ein neuer Ladevorgang begonnen hätte, behält aber die überschriebenen Ladelimits bei. Mit dieser API und [`charge_limits/override_duration`](/interfaces/mqtt_http/api_reference/charge_limits.mdx#charge_limits_override_duration) und [`charge_limits/override_energy`](/interfaces/mqtt_http/api_reference/charge_limits.mdx#charge_limits_override_energy) kann zu einem beliebigen Zeitpunkt ein neues "absolutes" Ladelimit gesetzt werden.
**Leerer Payload. Es muss einer der folgenden Werte übergeben werden: `null`, `""`, `false`, `0`, `[]` oder `{}`**
:::info[Beispiel]
    <Tabs groupId="apiType" queryString className="hidden-tabs">
        <TabItem value="http" label="HTTP (curl)">
```bash
# $HOST z.B. warp-AbCd
```
#### Schreiben
```bash
curl http://$HOST/charge_limits/restart -d 'null'
```



        </TabItem>
        <TabItem value="mqtt" label="MQTT (mosquitto)">
```bash
# $BROKER z.B. my_mosquitto.localdomain
# $PREFIX z.B. warp/AbCd
```
#### Schreiben
```bash
mosquitto_pub -h $BROKER -t $PREFIX/charge_limits/restart -m 'null'
```
