# Stromzähler
WARP Charger und Energy Manager unterstützen mehrere Stromzähler.
    Die Stromzähler-API ist aufgeteilt in einem allgemeinen Teil unter `meters/...`
    und einen Stromzähler-spezifischen Teil unter `meters/X/...` wobei `X` einer Stromzähler-Nummer entspricht.
    Der Stromzähler-spezifische Teil beinhaltet immer
    - [`meters/X/value_ids`](/mqtt_http/api_reference/meters.mdx#meters_X_value_ids)
    - [`meters/X/values`](/mqtt_http/api_reference/meters.mdx#meters_X_values)
    - [`meters/X/live`](/mqtt_http/api_reference/meters.mdx#meters_X_live)
    - [`meters/X/history`](/mqtt_http/api_reference/meters.mdx#meters_X_history)

    Beispielsweise können die Messwerte des ersten Stromzählers (der die Stromzählernummer 0 besitzt) unter `meters/0/values` gelesen werden.
    Außerdem hat jeder Zähler die APIs
    - [`meters/X/config`](/mqtt_http/api_reference/meters.mdx#meters_X_config)
    - [`meters/X/state`](/mqtt_http/api_reference/meters.mdx#meters_X_state)
    - [`meters/X/errors`](/mqtt_http/api_reference/meters.mdx#meters_X_errors)

    deren Inhalt von der Klasse des Stromzählers abhängt.
    Die Stromzählerklasse wird als das <a href="#unions_section">Union</a>-Tag von [`meters/X/config`](/mqtt_http/api_reference/meters.mdx#meters_X_config) angegeben.

    Jeder Stromzähler meldet seine Messwerte unter [`meters/X/values`](/mqtt_http/api_reference/meters.mdx#meters_X_values) als ein Array von Floats.
    Welcher Messwert wie zu interpretieren ist, kann unter [`meters/X/value_ids`](/mqtt_http/api_reference/meters.mdx#meters_X_value_ids) (einem Array von Ints, den sogenannten MeterValueIDs) ausgelesen werden:
    Wenn beispielsweise an Index 3 in [`meters/X/value_ids`](/mqtt_http/api_reference/meters.mdx#meters_X_value_ids) die MeterValueID 13 gelesen wird, dann ist der [`meters/X/values`](/mqtt_http/api_reference/meters.mdx#meters_X_values)-Wert an Index 3 als der Phasenstrom auf L1 zu interpretieren.

    Alle MeterValueIDs sind auf Github dokumentiert: [Liste aller MeterValueIDs](https://github.com/Tinkerforge/esp32-firmware/blob/master/software/src/modules/meters/meter_value_id.csv)

## `meters/X/history` {#meters_X_history_warp1}
<Tabs groupId="apiType" queryString className="hidden-tabs">

Eine 48-Stunden-Historie der Ladeleistung des X. Stromzählers in Watt. Bisher fehlende Werte werden durch null angezeigt. Die Historie wird von hinten nach vorne gefüllt, sodass null-Werte nur geschlossen am Anfang des Arrays auftreten, falls der ESP innerhalb der letzten 48 Stunden neugestartet wurde. [4, { "display_name": "API-Zähler", "value_ids": [13, 17, 21, 39, 48, 57] }]

der pro Phase Strom (Bezug + Einspeisung) und Wirkleistung (Bezug - Einspeisung) misst.
4 ist das Union-Tag des API-Stromzählers.

```bash
curl http://$HOST/meters/1/config -d '[4, { "display_name": "API-Zähler", "value_ids": [13, 17, 21, 39, 48, 57] }]'
```
